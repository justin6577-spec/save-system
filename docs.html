SaveSystem — Documentation

SaveSystem is a production-ready save/load plugin for Unreal Engine 5.4–5.8. It provides a clean, Blueprint-friendly API for managing game saves with multiple slots, metadata, screenshots, and compression.

Installation

  1. Copy the SaveSystem/ folder into your project's Plugins/ directory.
  2. Add "SaveSystem" to your .uproject Plugins array (or enable in Editor → Plugins).
  3. Regenerate project files (right-click .uproject → Generate Visual Studio project files).
  4. Build your project.
Requirement: UE 5.4 or newer. No external dependencies.

Quick Start (Blueprint)

  1. Open any Blueprint → right-click → search "Save Manager".
  2. Call Get Save Manager (returns the subsystem).
  3. Use Save Game To Slot / Load Game From Slot.
  4. Optional: call Get All Save Slots to list available saves with metadata and screenshots.

Quick Start (C++)

#include "SaveManager.h"

// Get the subsystem
USaveManager* SaveMgr = GetGameInstance()->GetSubsystem<USaveManager>();

// Save
FSaveGameData Data;
Data.PlayerName = TEXT("Hero");
Data.Level = 5;
SaveMgr->SaveGameToSlot(TEXT("Slot1"), Data);

// Load
FSaveGameData Loaded;
if (SaveMgr->LoadGameFromSlot(TEXT("Slot1"), Loaded))
{
    UE_LOG(LogTemp, Log, TEXT("Loaded player: %s, Level: %d"), *Loaded.PlayerName, Loaded.Level);
}

// Delete
SaveMgr->DeleteSlot(TEXT("Slot1"));

Features

Multiple Save Slots

Unlimited named save slots. Each slot stores game data + metadata + screenshot.

Save Metadata

Every save stores:

  • Slot name
  • Timestamp (UTC)
  • Playtime in seconds
  • Current level/map name
  • Player-defined display name

Screenshot Capture

Automatically captures a 256×256 screenshot when saving. Stored as PNG alongside save data. Use Get All Save Slots to retrieve thumbnails for UI.

Data Compression (Zlib)

All save data is compressed with Zlib before writing to disk. Reduces file size by 60–80%.

Async Operations

Saves are async by default. Use the callback/delegate variants to run logic after completion.

Save Validation

Built-in validation checks file integrity on load. Corrupted saves return failure gracefully.

Blueprint Support

Full BlueprintCallable API. All functions tagged with categories for easy discovery.

API Reference

USaveManager

// Save game to named slot (async)
void SaveGameToSlot(const FString& SlotName, const FSaveGameData& Data);

// Load game from named slot
bool LoadGameFromSlot(const FString& SlotName, FSaveGameData& OutData);

// Delete a save slot
bool DeleteSlot(const FString& SlotName);

// Get metadata for all save slots
TArray<FSaveSlotInfo> GetAllSaveSlots() const;

// Check if a slot exists
bool DoesSlotExist(const FString& SlotName) const;

// Get total number of save slots
int32 GetSlotCount() const;

// Set custom save directory (default: ProjectSaved/SaveGames/)
void SetSaveDirectory(const FString& Path);

// Configure auto-save interval (seconds, 0 = disabled)
void SetAutoSaveInterval(float IntervalSeconds);

FAQ

Q: Where are saves stored?
%PROJECT%/Saved/SaveGames/ by default. Configurable via SetSaveDirectory().

Q: Does it support cloud saves?
The plugin handles local save/load. Cloud sync can be added by implementing ISaveSystemPlatform (see source).

Q: Can I extend FSaveGameData?
Yes — it's a USTRUCT. Add your own fields or subclass as needed.

Q: Does this work in multiplayer?
Saves are local per-client. For server-authoritative saving, call Save/Load on the server and replicate data as needed.
