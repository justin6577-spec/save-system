DialogueSystem — Documentation

Branching dialogue trees with conditions, player responses, and events. Data-driven — define dialogues in DataTables, run them with a single ActorComponent.

Installation

  1. Copy DialogueSystem/ into Plugins/
  2. Add "DialogueSystem" to .uproject Plugins array
  3. Regenerate project files + build
Requirement: UE 5.4+. ActorComponent — add to any actor.

Quick Start (BP)

Open any Blueprint, search for the subsystem/component, and call the main API functions. All functions are BlueprintCallable.

Quick Start (C++)

#include "DialogueSystem.h"
// Get the subsystem/component and call Start/Initialize functions

Features

  • DataTable-driven dialogue trees
  • branching player responses
  • condition-gated response visibility
  • auto-advance nodes
  • dialogue events
  • condition system (SetCondition/GetCondition)
  • 3 Blueprint events
  • single ActorComponent — add to any actor

API

Full BlueprintCallable API. All public functions exposed to Blueprints.

Technical

  • Engine: 5.4–5.8
  • Platforms: Windows, Linux, Mac
  • Module: DialogueSystem (Runtime)
  • Classes: 5 (UDialogueComponent, FDialogueNode, FDialogueResponse, FDialogueCondition, FDialogueSnapshot)
  • Network: No
  • Blueprints: 0 (BlueprintCallable API)
